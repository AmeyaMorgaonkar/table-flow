# Coding Style

Always on. Apply to every file written or modified.

- Single responsibility per function and component — if it does two things, split it
- No magic numbers or strings — use named constants or enums
- Explicit over implicit — prefer clear variable names over clever one-liners
- No dead code — remove unused imports, variables, functions immediately
- Early returns over deeply nested conditionals
- Functions over 40 lines should be split
- No `any` type in TypeScript — always type explicitly
